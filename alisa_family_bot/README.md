# AlisaUniverseBot MVP (етап 1)

## Запуск локально

```bash
cd alisa_family_bot
python3.11 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
python -m app.main
```

Якщо `WEBHOOK_BASE_URL` не заданий, бот запускається в polling режимі.

## Доступ до каналу

- Основний сценарій: користувач натискає `Join` у закритому каналі.
- Бот отримує `chat_join_request`, створює заявку і надсилає її обом супер-адмінам.
- Після підтвердження адміном бот викликає `approve_chat_join_request` і користувач отримує доступ.
- `/start` також підтримується як резервний вхід у той самий pending-флоу.

## Відкладені пости

- Кнопка `🆕 Новий пост` підтримує публікацію `зараз` або `згодом`.
- Підтримується одне відео або кілька фото (альбом) в одному пості.
- Відкладені пости зберігаються в PostgreSQL (`scheduled_posts`), тому не губляться після рестарту бота.
- Після рестарту воркер автоматично підхоплює pending-задачі та публікує їх у час `publish_at`.

## Памʼятні моменти (етап 2)

- Супер-адмін додає `✨ Додати памʼятний момент`.
- Підтримується текст, одне відео або кілька фото (альбом).
- Момент одразу публікується в канал і автоматично розсилається активним користувачам у приватні чати.
- Доступні реакції `❤️ 👍 👏`, збереження в `milestone_reactions` та `reactions`.

## Підтвердження підписки

- Після підтвердження нового учасника в канал надсилається welcome-пост із роллю та імʼям.
- У welcome-повідомленні є підказка перейти в бота і натиснути `/start` для приватних сповіщень.
- Для ролі доступний fallback: ручний ввід ролі адміном.

## Календар розвитку

- Кнопка `📈 Календар розвитку`: вага, зріст, події (перший зуб/кроки/місячний звіт).
- Після фіксації дані зберігаються і публікуються в канал.
- Для ваги/зросту бот показує приріст від попереднього виміру.
- Бот надсилає нагадування адмінам:
  - вага: раз на тиждень
  - зріст: раз на місяць

## Донати і бейджі (етап 2)

- Під кожним фото/памʼятним моментом є кнопка `Підтримати Алісу 🍼` (посилання з `DONATION_URL`).
- Кнопка `✅ Я задонатив` дозволяє вручну зафіксувати суму донату в БД (`donations`).
- Поле `users.total_donated` оновлюється автоматично.
- Бейджі оновлюються за активністю (реакції/донати).
- У приватному чаті є кнопки `👤 Мій профіль` та `🏆 Рейтинг донаторів`.
- Щомісяця бот формує топ-3 донаторів без публікації сум донатів.

## Хештеги

- Для нових постів і памʼятних моментів хештеги додаються автоматично.
- Можна додати власні хештеги під час створення памʼятного моменту.

## Railway (webhook)

1. Створіть PostgreSQL plugin.
2. Додайте ENV:
   - `BOT_TOKEN`
   - `DATABASE_URL`
   - `CHANNEL_ID`
   - `SUPER_ADMINS`
   - `DONATION_URL` (наприклад `https://send.monobank.ua/your-link`)
   - `DONATION_CARD` (опційно)
   - `WEBHOOK_BASE_URL` (наприклад `https://your-app.up.railway.app`)
   - `WEBHOOK_PATH` (наприклад `/webhook`)
   - `WEBHOOK_SECRET` (випадковий довгий рядок для валідації webhook)
3. Стартова команда:

```bash
python -m app.main
```

4. Healthcheck endpoint для Railway:

```text
GET /health
```
